## Что ещё стоит сделать

### 1. Отключить superres и film grain
```c
// src/decode.c — при парсинге frame header
// superres пересчитывает размеры и добавляет upscale pass
// при 64px это холостая работа если не используется
```
Если стрим без film grain — можно закомментировать `film_grain_synthesis()` в `picture.c`.

---

### 2. Loop Restoration Filter (LRF) — следующий кандидат на вырез
После CDEF идёт LRF (Wiener / Self-Guided). На 64px он работает по блокам 64×64 — то есть **один единственный блок на весь кадр**, но вычислительно дорогой. Смотри `src/lf_apply.c` и `src/lr_apply.c`:

```c
// В decode.c найди вызов:
dav1d_lr_apply(f, p);
// Можно условно скипать если width <= 64 && height <= 64
```

---

### 3. Убрать ref frame motion vector storage
При таком разрешении TMVP (Temporal Motion Vector Prediction) экономит мало, но хранение и обход MV-буферов по всему кадру — лишняя работа. В `src/refmvs.c` — `dav1d_refmvs_save_tmvs()`.

---

### 4. Уменьшить tile/frame threading overhead
Ты уже однопоточный, но убедись что:
```c
// src/lib.c при создании контекста:
s.n_threads = 1;
s.max_frame_delay = 1;
```
И что tile worker pool не создаётся вообще — смотри `dav1d_open()`, там есть инициализация `task_thread`.

---

### 5. Bitdepth — оставь только нужный
dav1d компилируется с шаблонами под 8bpc и 16bpc. Если твой стрим всегда 8-bit:
```makefile
# meson_options.txt или CLI:
-Dbitdepths=8
```
Это убирает половину кодовых путей в `src/mc_tmpl.c`, `src/ipred_tmpl.c` и т.д.

---

### 6. Chroma subsampling — если можешь контролировать стрим
При 64×64 с 4:2:0 chroma planes — 32×32. Если перейти на 4:0:0 (grayscale) — MC и предсказание для chroma вообще не нужны.

---

### 7. Профилировщик покажет главное
На generic без ASM узкие места будут в:
- `dav1d_mc_avg` / `dav1d_mc_w_avg` — усреднение ref frames
- `dav1d_ipred_*` — intra prediction
- `inv_txfm` — обратное преобразование

Для 64px кадра количество блоков мизерное, поэтому **decode overhead** (парсинг bitstream, OBU, tile) может составлять значимую долю. Смотри через `perf stat` или `gprof`.

|---|---|---|
